구현해야 하는 기능 정리 및 고찰

→ 캐릭터와 주요 장면 이미징 처리

- 유저가 원하는 캐릭터 이미지로 생성하기 → 그림체를 프롬프트화 UI에서 유저가 선택 가능하게 만들거나 유저가 직접 이미지를 업로드(이미지 갤러리 접근 기능)
- 매 번 이미지를 생성하는 것은 토큰 부담이 심하므로 어떤 장면에서 이미지를 생성시킬 것인지 감정의 고조화, 스토리 내 중요도와 같은 기준으로 스코어를 측정해 특정 값 이상 일 경우 이미지 출력하기
- 주요 캐릭터들의 성장 배경, 성격, 프로필등을 생성하여 캐릭터의 일관성 부여(RAG)
- 주요 장면에서 배경화면 자동 전환 기능(분위기를 나타내줄수있는 장소와 같은 이미지, 실사X)

→ 특정 스토리 진입시 배경음악 재생 기능

- 분위기 별로 배경음악을 학습한 모델을 선정해 유저가 해당 페이지 진입시 음악 재생(저작권X)
- 오디오 아이콘 클릭을 통해 mute 토글 기능

→ **오프라인 모드 및 캐싱 기능**

- 서버에서 매번 텍스트와 이미지를 불러오지 않고 로컬DB 활용 및 캐싱 로직 추가

실시간 AI 대화형 서비스를 위한 UI/UX

- 하단 고정 입력창 → 대화가 필요한 타이밍과 스토리 전개때를 구분해서 입력창 애니메이션화
- 캐릭터의 감정 상태를 나타내는 플로팅 카드를 상단에 배치, 클릭 이벤트 후 정보를 확인 가능한 UI구현`Lucide Icons` , `Material Symbols`  등 한가지로 일관성 유지
- 소설을 한번에 화면에 나타내지 않고 타이핑 효과로 몰입감 향상
- AI와 관련된 기능 사용시 응답을 기다리는 구간을 로딩 스피너가 아닌 다른 UI로 고민해볼 것
- 독서 서비스에 알맞는 폰트들을 찾아볼 것
